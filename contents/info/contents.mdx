        >### React
        >**확장성 있고 유지보수가 용이한 컴포넌트**를 만드는데 큰 관심이 있습니다. 이를 위해 **뷰와 비즈니스 로직을 분리**할 수 있으며 분리된 로직을 상황에 따라 **커스텀 훅이나 유틸 함수**로 분리할 수 있습니다.
        >
        >필요한 상황에서만 `useEffect`를 사용하고자 합니다. 이를 위해 컴포넌트가 재 랜더링이 트리거되는 상황에 대해 알고 있습니다.
        >
        >초기 로딩 속도를 개선하기 위해 [Dynamic import, lazy loading과 같은 기능을 이용](https://github.com/SobiSa-Expense-Hunter/SobiSa/blob/7d0375f4e3b64ca2011a64a64ed56890c93d3538/src/components/list/index.tsx#L20-L21 )해 최적화 할 수 있습니다.

        >### Next.js
        >[Next js API Route를 사용할 수 있습니다.](https://github.com/SobiSa-Expense-Hunter/SobiSa/blob/develop/src/pages/api/products.ts)
        >빠른 데이터 패칭을 위해 [`getStaticPaths`, `getStaticProps` 과 같은 Next.js Server Side API*를 이용한 Data Fetching을 구현해 본 경험](https://github.com/Seo0H/seo0h.github.io/blob/027e730aa972a65287b172a4deb08aa4b8b73dea/src/pages/index.tsx#L16-L27)이 있습니다.
